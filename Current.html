AWS

I took an Introduction to Cloud Fundamentals course and learned the basics of creating a cloud environment within AWS. For our final project, we had to create a full deployment and calculate how much it would cost to keep it up and running for a year.

The requirements were as follows:

  1. Show that you have MFA enabled on your root account.
  2. Create a storage container or bucket that contains a static website.
  3. Create a VPC, VNet, or subnet with Internet access.
  4. Create a second VPC, VNet, or subnet that does NOT have access to the Internet. Make sure the IP address spaces for these two networks do not overlap.
  5. Create a peering connection between them.
  6. Deploy a VM in each VPC, VNet, or subnet so that one has access to the Internet and the other one does not.
  7. Use a drawing app, such as Visio or app.diagrams.net, to draw a network diagram showing the VPCs/VNets/subnets and their resources, with logical connections, CIDR ranges, gateways, firewall rules, and relative position of each resource.
  8. Make sure both VMs can send and receive pings.
    • For each VM, ping a location on the Internet (such as Google's DNS server at 8.8.8.8) to prove one VM has access to the Internet and the other one does not.
  9. Apply an alert or alarm, or enable automatic updates for one of the VMs.
  10. Use your cloud platform's pricing tools to calculate the cost of your deployment for one year.
  11. Fully document your entire deployment.
